# Glossário

[Início](../README.md) · [Checklist](07-checklist-de-integracao.md) · [Exemplos](exemplos-de-integracao.md)

| Termo | Significado neste guia |
|---|---|
| agente ou especialista | conjunto de instruções e limites para uma categoria de atendimento |
| analytics | agregações usadas para acompanhar comportamento e resultado |
| classificação | transformação de texto livre em categoria estruturada |
| contato | identidade conhecida que pode possuir várias sessões |
| contexto | informações selecionadas para apoiar uma chamada ao modelo |
| contrato | formato esperado de entrada ou saída entre partes do sistema |
| CRM | organização do histórico e dos sinais de relacionamento |
| evento | registro de algo relevante que aconteceu no fluxo |
| fallback | comportamento seguro quando o caminho principal falha |
| gateway de IA | camada que esconde diferenças entre provedores e SDKs |
| insight | interpretação derivada de métricas ou conversas |
| intenção | categoria que representa a necessidade principal |
| metadado | dado que descreve uma mensagem ou sessão |
| payload | conteúdo estruturado transportado em um evento ou chamada de API |
| provedor | serviço que disponibiliza um modelo de linguagem |
| rate limit | limite de chamadas para reduzir abuso, custo e instabilidade |
| score | indicador explicável usado para ordenação ou priorização |
| SDK | biblioteca oficial ou cliente usado para falar com um serviço externo |
| sessão | unidade de atendimento composta por mensagens e eventos |
| triagem | etapa que identifica intenção e destino recomendado |
